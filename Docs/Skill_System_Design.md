# 스킬 시스템 디자인

> **Note**: 세로로 UI가 바뀐다고 생각하면 된다.

## 1. 시스템 개요

이 시스템은 **AI 로봇의 진화 과정을 공간 기반 강화 트리로 표현**한다.

각 강화 노드는 2D 격자(X, Y 좌표)에 배치되며, **인접한 노드(상·하·좌·우)**를 해금해야 다음 강화를 진행할 수 있다.

핵심 목표는 다음과 같다.

- 플레이어가 **“어떤 방향으로 AI를 발전시킬지”** 직관적으로 선택하게 만들 것
- 단순 수치 누적이 아니라, **구조적 성장(자동화 → 병렬 → 특이점)**을 체감하게 할 것
- 중앙은 안정적·기본적, 외곽으로 갈수록 **극단적·강력·리스크 있는 강화**가 위치하도록 할 것

---

## 2. 노드 구조 정의

각 강화 노드는 아래 필드를 가진다.

| 필드 | 설명 |
| --- | --- |
| ID | 고유 식별자 (예: AUTO01, GPUR07) |
| Name | UI에 표시될 강화 이름 |
| Desc | 강화 효과 설명 |
| Note | 내부 메모 또는 특수 조건 |
| XCoord, YCoord | 트리 상 좌표 |
| isFirstNode | 시작 노드 여부 (1이면 시작 가능) |

### 규칙 요약

- 모든 좌표는 **정수 좌표**
- 대각선 연결은 인정하지 않음
- 강화 가능 조건: **이미 해금된 노드와 맨해튼 거리 1**

---

## 3. 강화 카테고리와 의미

강화는 “기능적 성격”에 따라 묶이며, 이 성격이 **배치 방향**과 **효과 성향**을 결정한다.

### 3.1 중앙 계열 (Core)

대표: `AUTO`, `CPUR`, `MATR`

- 특징: 안정적, 기본 효율 상승
- 배치: 원점(0,0) 근처
- 효과 예:
    - 자동 터치 증가
    - 기본 공격력/방어력 소폭 증가
- 설계 의도:
    - 초반 접근성
    - 어떤 빌드로 가든 반드시 거치는 “AI 기초 인프라”

---

### 3.2 자동화 / 경제 계열 (Left 방향)

대표: `AUTO`, `CBKD`, `RLOP`

- 특징: 생산성, 반복 제거
- 효과 예:
    - 초당 자동 터치 증가
    - 골드 획득량 증가
    - 업그레이드 비용 감소
- 설계 의도:
    - “AI가 일을 대신한다”는 테마를 가장 먼저 체감
    - 클릭 → 시스템화로 넘어가는 첫 단계

---

### 3.3 인지 / 정확 계열 (Up 방향)

대표: `SENS`, `VALG`, `CALB`, `KNAG`

- 특징: 명중률, 범위, 상황 대응
- 효과 예:
    - 공격 범위 증가
    - 적 회피 확률 감소
    - 미클리어 스테이지 보정
- 설계 의도:
    - “더 똑똑해지는 AI”
    - 단순 화력보다 **판단 능력 강화**

---

### 3.4 생존 / 안정 계열 (Down 방향)

대표: `HERR`, `DCOL`, `WBOX`, `ACTU`

- 특징: 방어, 실수 감소, 이동성
- 효과 예:
    - 크리티컬 실패 방어
    - 피격 크리티컬 확률 감소
    - 이동속도 증가
- 설계 의도:
    - 장기 플레이 안정성
    - 자동화·화력 빌드의 리스크 보완

---

### 3.5 화력 / 병렬 계열 (Right 방향)

대표: `PARR`, `OFIT`, `FINT`, `DFIR`

- 특징: 공격 구조 자체를 바꿈
- 효과 예:
    - 로봇 팔 수 증가 (대신 개별 화력 감소)
    - 특정 스탯 극단 증폭
    - 반복 스테이지 특화
- 설계 의도:
    - “AI가 동시에 여러 일을 처리”
    - 수치 상승이 아니라 **공격 방식의 변화**

---

### 3.6 외곽 / 특이점 계열

대표: `GPUR`, `TPUR`, `MATT`, `SGRL`

- 특징: 매우 강력하지만 접근 비용 큼
- 효과 예:
    - 최종 공격력 대폭 증가
    - 즉사 확률
    - 고급 하드웨어 도입
- 설계 의도:
    - 엔드게임 빌드
    - “AI가 인간을 넘어서는 순간”

- n-1번이 n번보다 비용이 높지 않을 것
